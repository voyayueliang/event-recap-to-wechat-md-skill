# AI Writing Method Playbook

Use this reference when turning event material into public writing. It translates Moon's content workbench experience into concrete writing moves.

## 1. Build A Material Ladder

Before drafting, sort material from concrete to abstract:

1. `场景`: room, object, photo, gesture, audience question, whiteboard, slide, table, chat moment
2. `原话`: short quotes, repeated phrases, questions, surprising wording
3. `行为`: who came, asked, replied, stayed, introduced, shared, signed up, followed up
4. `事实`: date, place, organizer, partner, agenda, speaker names, product names, public links
5. `观察`: what the organizer noticed in the room
6. `判断`: what this event revealed
7. `行动`: what readers/partners/community members should do next

Draft from the lower rungs upward. If a paragraph begins at `判断`, ask which `场景/原话/行为` should come first.

## 2. Find The Opening Scene

Do not open with "本次活动圆满举行", broad trends, or institutional self-praise unless the user explicitly wants official copy.

Good opening candidates:

- a sentence the organizer kept thinking about after the event
- a concrete audience question
- a scene from arrival, discussion, Q&A, demo, break, or closing
- an image that reveals the room's energy
- a contrast between official framing and what the room actually cared about
- a small tension: what was expected vs what emerged

Opening test:

```markdown
If someone did not attend this event, can they still see one specific moment after reading the first 3 paragraphs?
```

If not, rewrite the opening from a scene.

## 3. Extract The Organizer's Judgment

The user's口述复盘 is the editorial backbone. Listen for:

- "我后来一直在想..."
- "最抓我的不是..."
- "我没想到..."
- "我其实有点警惕..."
- "这场活动让我看到..."
- "下次应该..."

Turn these into one cautious thesis. Do not inflate them into universal claims.

Safer wording patterns:

- "这场活动至少提醒我..."
- "对我来说，更重要的不是..., 而是..."
- "如果只从现场看，一个很明显的问题是..."
- "这还不是一个完整答案，但它留下了一个问题..."

## 4. Use Public Reports Without Becoming Derivative

Public reports can do four jobs:

- verify facts
- reveal what angle has already been over-written
- provide public context
- offer contrast with the organizer's field observation

Do not copy their structure or lead. If a report's wording is distinctive, attribute it. If it only confirms a basic fact, use it as fact-checking.

Rewrite pattern:

```markdown
公开报道已经把 [官方事实/行业语境] 说清楚了。但从这场活动的现场看，更值得记录的是 [现场问题/人群反应/组织者判断]。
```

Use sparingly. The article should still belong to the organizer.

## 5. Turn "Successful Event" Into Evidence

Avoid unsupported success language:

- "反响热烈"
- "干货满满"
- "圆满成功"
- "深度链接"
- "有效赋能"

Replace with observable evidence:

- audience stayed after closing to ask questions
- a participant repeated a phrase
- someone asked for resources or a next meeting
- partner shared a concrete follow-up
- a question appeared across multiple groups
- a photo/whiteboard shows what people worked on

Template:

```markdown
与其说这场活动"反响热烈"，不如说现场留下了三个可观察的信号: ...
```

Use "signal" only when the evidence is concrete.

## 6. Write Images Into The Argument

For each image, assign one job:

- `建立场景`: show the room, table, stage, or setup
- `证明互动`: show Q&A, group work, demo, discussion, or artifact
- `保留人物`: show speaker/organizer/partner presence with consent
- `解释方法`: show whiteboard, notes, slide, worksheet, materials
- `承接 CTA`: show QR code/resource/next event only when intended

Write one sentence before or after the image that tells readers why it appears.

Bad:

```markdown
![活动现场](IMAGE_01)
```

Better:

```markdown
讨论最密集的时候，大家围在同一张白板前，把问题从"怎么传播"改写成了"谁真的需要这个答案"。

![图1: 参与者围绕白板讨论问题定义](IMAGE_01)
*图注: 这一轮讨论留下了后续工作坊最重要的问题。*
```

## 7. Keep Dialogue When Dialogue Is The Value

Do not over-synthesize when:

- the guest's words are sharper than summary
- the organizer's question reveals the method
- disagreement or hesitation is the real value
- the event was exploratory

Use edited dialogue, Q&A, quote-led sections, or conversation digest.

## 8. Build Sections By Function

Each section should have a job:

- `开场`: show the event's live question
- `事实`: orient readers without press-release padding
- `现场`: make the room visible
- `判断`: say what the event revealed
- `伙伴`: show contribution and attribution
- `方法`: recover reusable practice
- `下一步`: invite a specific action

If two sections do the same job, merge them. If a beautiful paragraph does not serve any job, move it to a素材卡.

## 9. Rewrite AI-Taste Paragraphs

When a paragraph sounds too complete:

1. Identify the abstract claim.
2. Ask what scene, quote, behavior, or image supports it.
3. Move the concrete material before the claim.
4. Weaken unsupported certainty.
5. Keep one unresolved question if the source is exploratory.

Before:

```markdown
这不仅是一场关于 AI 的活动，更是一场关于未来教育生态的深度共创。
```

After:

```markdown
活动结束后，有一个问题还留在白板上: 当孩子已经可以用 AI 做出作品，老师和家长到底应该看什么？

这可能比"怎么教 AI 工具"更接近这场讨论的核心。
```

## 10. End With A Real Next Action

Avoid generic endings such as "期待未来继续同行".

Choose one:

- next event or registration
- reply with a question
- resource link
- partner thanks with specific contribution
- sample call
- invitation to share cases
- internal follow-up note if not public

The CTA should match the event's real purpose and evidence level.
