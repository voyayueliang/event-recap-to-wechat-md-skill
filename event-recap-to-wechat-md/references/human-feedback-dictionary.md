# Human Feedback Dictionary

Use this reference whenever the user reacts to a draft in natural language. Treat feedback as workflow information, not only style preference.

## Feedback Map

| User feedback | Likely meaning | Do this |
| --- | --- | --- |
| `太 AI 味` | abstract, over-smooth, too many template shells, no living scene | remove slogans, move scenes/quotes forward, weaken conclusions, vary rhythm |
| `不像我` | wrong creator position, wrong first person, too corporate/teacherly/influencer-like | restore Moon's question, hesitation, taste, and field relationship |
| `太完整 / 太顺` | AI closed loops that should remain open | remove over-complete takeaways, keep uncertainty, turn some conclusions into questions |
| `不要写成这种内容` | output form is wrong | reconsider: article, edited dialogue, content view, material card, social post, internal memory |
| `逻辑像拼贴` | sections do not serve one main argument | write one-sentence thesis, assign each section a job, remove pretty fragments |
| `前面太轻，后面太重` | opening lacks material/logic; later sections carry too much | add scene, quote, context, or stakes to the opening; do not only cut later text |
| `标题不对` | reader problem or event tension is not named | generate title groups: scene, question, direct judgment, partner/community angle |
| `太像新闻稿` | official phrasing is dominating field texture | replace "圆满/热烈/成功" with observable evidence and organizer judgment |
| `太营销` | CTA or value claim is over-selling | reduce claims, return to evidence, make CTA softer and more specific |
| `有点敏感` | privacy, relationship, consent, partner, client, or identity risk | pause, mark `发布前确认`, anonymize, weaken, or move to internal memory |
| `这句不能这么说` | fact, stance, or relationship boundary problem | delete or rewrite weaker; do not defend the sentence |
| `嘉宾声音没了` | synthesis erased speaker texture | restore short quotes, Q&A, speaker labels, or edited dialogue format |
| `图片不对` | image does not support narrative or has permission risk | revise image job, move/caption/replace/flag authorization |
| `太长` | repeated explanation, too many side branches | keep main arc, remove duplicate interpretation, move extras to asset cards |
| `太短` | missing scenes or material anchors | add source-backed scene/quote/transition; do not pad with generic summary |
| `想要更能发` | needs clearer output contract | ask/choose platform and produce copy-ready version with publish checks |

## Revision Protocol

When feedback arrives:

1. Identify which layer failed:
   - output form
   - source/evidence
   - logic
   - voice
   - boundary
   - platform fit
   - length/rhythm
2. Name the likely issue briefly.
3. Apply the matching action.
4. Re-run the relevant quality gates.
5. Keep unresolved items under `发布前确认`.

Do not rewrite the whole piece if the problem is local. Do not make local edits if the output form is wrong.

## High-Priority Feedback

These require slowing down:

- `不对劲`
- `不像我`
- `敏感`
- `不能这么说`
- `这个人不能被这样写`
- `这个不是我要的形式`

These usually mean judgment, relationship, or format is wrong. Ask only if needed; otherwise produce a safer version plus a confirmation list.

## Title Feedback

When the title is wrong, produce groups:

```markdown
## 标题候选

### 现场张力

### 读者问题

### 直接判断

### 伙伴/社区角度

### 轻传播
```

Avoid clickbait if the source is reflective or relationship-sensitive.

## Boundary Feedback

When the user marks something sensitive, choose one:

- remove
- anonymize
- paraphrase
- weaken the claim
- change public article into internal memory
- ask for permission
- move to `发布前确认`

Never solve sensitivity by making wording prettier.
