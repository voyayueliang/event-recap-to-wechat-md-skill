# Output Contracts

Use this reference to choose and format deliverables. The goal is to make the skill operational for event organizers, not only good at writing one article.

## Contract Selection

| User need | Contract |
| --- | --- |
| "直接出公众号 md" | WeChat Markdown Draft |
| "活动结束 24 小时内发出去" | 24-Hour Recap Pack |
| "给社区留下记录" | Community Memory Pack |
| "给伙伴/赞助方看" | Partner/Sponsor Pack |
| "这场活动生成了公共知识" | Public Learning Pack |
| "整理成可复用素材" | Asset Recovery Card |
| "发布后看反馈" | 72-Hour Feedback Recovery |
| "先判断战略叙事/组织定位/伙伴风格" | Strategic Narrative Diagnostic |
| "调研已有报道/其他参与方/小红书或社媒体验" | Public Material Research Map |
| unclear or risky material | Content View / Gap Card |

## Content View

Use before drafting when source material is long, messy, sensitive, or form is unclear.

```markdown
## 内容查看台

- 这份素材最适合做什么:
- 最强公共问题:
- 最值得保留的现场:

- 推荐输出:
- 公开边界:
- 还缺什么:
- 下一步:
```

## Public Material Research Map

Use before drafting when the user wants to integrate existing online material: official reports, other participant recaps, media coverage, partner posts, Xiaohongshu/Rednote notes, social posts, comments, public attendee experiences, or project/work coverage.

This output should help the user decide what to use. Do not let the agent make final interpretive judgments for the user.

```markdown
## 公开资料调研地图
```

## Final Contract Rule

If the selected contract is bigger than the user's need, shrink it. If the user asks for infrastructure but receives only one article, expand it.
