# Community Event Infrastructure

Use this reference when the event recap workflow is being positioned as infrastructure for communities, salons, workshops, conferences, brand events, learning groups, creator communities, or event organizers.

The product thesis: a good event should not disappear after the room closes. The workflow should turn event material into public memory, useful content, relationship follow-up, partner proof, and reusable assets.

## Positioning

Do not describe this as an AI writing tool. Describe it as:

- a post-event content desk
- an event memory and publishing layer
- an AI operating layer for community events
- a 24-hour post-event recap pack generator
- a source-aware workflow for community organizers

Core promise:

```text
Turn recordings, organizer debriefs, partner posts, public coverage, and selected images into a publishable recap plus reusable community assets.
```

## Best Fit

Use this infrastructure framing for:

- recurring community events
- salons, workshops, roundtables, and meetups
- brand or partner-led events
- creator/community education programs
- conferences and side events
- organizations with strong events but limited editorial capacity
- teams that need sponsor/partner proof without writing generic press releases

Weak fit:

- purely private gatherings
- events with unclear consent
- teams that only want viral posts
- events with no recording, notes, images, or organizer judgment

## Infrastructure Modes

### 24-Hour Recap Pack

Use when the organizer needs public output quickly.

```markdown
## 24 小时会后传播包

### 主稿
- 推荐标题:
- 摘要:
- 正文 md:

### 社交短内容
1.
2.
3.

### 图片使用建议

### 伙伴/嘉宾感谢语

### 发布前确认
```

### Community Memory Pack

Use when the event should become internal knowledge and future programming material.

```markdown
## 社区记忆包

### 这场活动留下的问题

### 现场场景/原话/图片

### 可回收素材卡
| 卡片 | 类型 | 用途 | 边界 |
| --- | --- | --- | --- |

### 下一场活动线索

### 关系跟进
```

### Partner Or Sponsor Pack

Use when partners, sponsors, co-hosts, or institutions need proof of contribution.

```markdown
## 伙伴/赞助方回顾包

### 伙伴贡献

### 公共价值

### 可引用素材

### 需要署名/确认

### 下一步合作线索
```

### Public Learning Pack

Use when the event generated useful public knowledge but not necessarily a polished article.

```markdown
## Public Learning Pack

### What happened

### What the room was really asking

### Useful quotes or scenes

### Open questions

### Source notes
```

## Organizer Input Contract

Ask for or infer:

- event name, date, city/platform, format
- organizer, partner, sponsor, speaker, attendee roles
- recording/transcript/notes/slides
- organizer debrief: what mattered, what surprised them, what should be remembered
- selected images and usage permission
- public reports, official recaps, partner posts, media links
- desired output: article, recap pack, partner pack, internal memory, or all
- public/private boundary
- CTA or next action

Do not block the workflow if some inputs are missing. Draft with assumptions and list gaps in `发布前确认`.

## Governance Rules

- No auto-publishing. Human approval is required.
- Treat consent, photo usage, speaker quotes, partner attribution, and sponsor wording as infrastructure-level requirements.
- Use public reports for verification and context, not as text to copy.
- Do not imply endorsement, sponsorship, partnership, or attendance beyond the sources.
- Do not turn attendee reactions into claims about community success without evidence.
- When privacy is unclear, output internal memory rather than public copy.

## Asset Recovery

Every substantial event should recover at least one asset:

- event article
- source cards
- quote cards
- scene cards
- FAQ or audience question list
- partner proof
- case note
- method note
- next-event topic
- offer/product signal
- follow-up message

The goal is compounding community memory, not one-off content.
